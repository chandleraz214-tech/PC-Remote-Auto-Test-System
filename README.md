# 📦 PC 远程自动测试系统
通过 Mac 浏览器远程控制华为电脑，实现全自动压测与物理唤醒

---
## 📖 项目简介
本系统是一套跨平台 PC 远程控制与自动化测试解决方案，通过 WebSocket + HTTP 通信协议，实现 Mac 电脑 与 华为电脑 之间的远程控制与自动化测试。

核心功能功能说明
- **🖥远程关机 / 重启 / 睡眠**：通过网页按钮一键控制，无需现场操作
- **🔌物理唤醒（触控开机）**：电脑关机状态下，触控按压电源键开机，实现完全无人值守
- **🧪一键启动 BurnInTest 压测**：支持 3 小时标准压测和 15 分钟自定义压测
- **📄自动获取压测报告**：解析 BIT_log.cert.htm 文件，提取 CPU/Memory/2D/3D Graphics PASS/FAIL
- **📊内存信息查询**：通过 WMI 获取内存信息、速度、序列号
- **🔄断线自动重连**：客户端与服务端断开后自动恢复连接，无需人工干预
- **⏰开机自启动**：华为客户端通过任务计划程序开机自动运行

#### 通信流程
用户点击按钮 → 浏览器 → Mac服务端(WebSocket) → 华为客户端 → 执行命令 → 返回结果 → 浏览器显示
#### 物理唤醒流程
用户点击唤醒 → Mac服务端(HTTP) → 舵机控制器API → 舵机按压电源键 → 华为电脑开机 → 客户端自动连接


| 组件 | 设备 | IP | 端口 | 说明
| :--- | :--- | :--- | :--- | :---
| 服务端 | Mac | 192.168.5.108 | 5001 (HTTP) / 8765 (WS) | 提供控制面板 + WebSocket 服务
| 客户端 | 华为 | 192.168.5.143 | 主动连接 | 执行命令，连接服务端
| 舵机 | 控制器 | 192.168.5.122 | 80 (HTTP) | 物理唤醒，按压电源键

## 🚀 快速开始

### 第 1 步：Mac 端部署
#### 1.1 将 server.py 放到桌面
（bash）
```bash
cd ~/Desktop
pip3 install flask requests websockets
nohup python3 server.py > server.log 2>&1 &
```
#### 1.2 安装依赖
（bash）
```bash
/usr/bin/python3 -m pip install flask requests websockets
```
#### 1.3 启动服务
（bash）
```bash
cd ~/Desktop
nohup /usr/bin/python3 server.py > server.log 2>&1 &
```
#### 1.4 验证运行状态
（bash）
```bash
ps aux | grep server.py | grep -v grep
# 应该看到 Python 进程

curl http://127.0.0.1:5001
# 应该返回 HTML 内容
```

### 第 2 步：Windows 客户端部署（华为电脑）
#### 2.1 环境要求
<ul>
  <li>Windows 10 / Windows 11</li>
  <li>.NET 8.0 SDK 或运行时</li>
  <li>Visual Studio 2022（推荐）或 VS Code + C# 扩展</li>
</ul>

#### 2.2 创建项目
<ol>
  <li>打开 Visual Studio 2022</li>
  <li>创建新项目 → 控制台应用 (.NET 8.0)</li>
  <li>项目名称：PCAutoTestClient</li>
  <li>将 Program.cs 代码粘贴到项目中</li>
  <li>将 PCAutoTestClient.csproj 配置粘贴到项目文件中</li>
</ol>

#### 2.3 安装 NuGet 包
在“管理 NuGet 程序包”中安装：

| 包名                 | 用途
|:-------------------|:----
| Newtonsoft.Json    | JSON 序列化/反序列化
| System.Management  | WMI 硬件信息查询

#### 2.4 修改配置
在 Program.cs 中找到以下代码并修改：

（csharp）
```csharp
private static string _serverUrl = "ws://192.168.5.108:8765";  // 改为 Mac 实际 IP
```

#### 2.5 编译运行
（bash）
```bash
dotnet build
dotnet run
```
或直接在 Visual Studio 中按 F5 运行。

#### 2.6 以管理员身份运行
<ul>
  <li>在 bin\Debug\net8.0\ 中找到 PCAutoTestClient.exe</li>
  <li>右键 → 以管理员身份运行</li>
</ul>

    ⚠️ 必须管理员身份运行，否则关机/重启/压测等操作会失败！

### 第 3 步：访问控制面板
在 Mac 浏览器中打开：

```text
http://192.168.5.108:5001
```
看到绿色状态条 “✅ 设备在线” 表示连接成功！

## 🔧 开机自启动配置（华为客户端）
### 使用任务计划程序
<ol>
  <li>按 Win + R，输入 taskschd.msc，回车</li>
  <li>右侧点击 “创建基本任务”</li>
  <li>名称：PCAutoTestClient</li>
  <li>触发器：“当计算机启动时”</li>
  <li>操作：“启动程序”</li>
  <li>程序：浏览选择 PCAutoTestClient.exe</li>
  <li>起始于：填写 exe 所在文件夹路径（如 D:\Projects\PCAutoTestClient\bin\Debug\net8.0\）</li>
  <li>点击 “完成”</li>
  <li>在任务列表中右键 → 属性</li>
  <li>常规 → ✅ “使用最高权限运行”</li>
  <li>条件 → ❌ 取消 “只有在计算机使用交流电源时才启动此任务”Fourth item</li>
</ol>

## ❓ 常见问题及解决方案
| 问题	             | 可能原因	         | 解决方法     
|:----------------|:--------------|:---------
| ❌设备离线           | 客户端未运行|启动PCAutoTestClient.exe（管理员身份）                
| ❌设备离线           | 客户端未以管理员身份运行|右键→以管理员身份运行                           
| ❌设备离线           | Mac 服务端未运行| 执行ps aux ｜ grep server.py检查，若未运行则重启
| ❌设备离线IP         | 地址变化| 确认 Mac IP，修改客户端 _serverUrl 重新编译                      
| ❌WebSocket连接失败  | 端口被占用|lsof -i :5001 / lsof -i :8765 查看占用进程并结束          
| ❌WebSocket连接失败  | 防火墙拦截|检查Mac/Windows防火墙设置，放行对应端口                      
| ❌压测未启动          | BurnInTest未安装|安装到C:\Program Files\BurnInTest\bit.exe 
| ❌报告未生成          | 报告目录无权限|确保D:\log\RAMTest\目录存在且可写入                    
| ❌报告解析全部Not found| 报告格式变化|检查BIT_log.cert.htm内容，调整正则表达式            
| ❌唤醒无效           | 舵机控制器不在线|检查192.168.5.122是否可访问                        
| ❌唤醒无效           | 按压参数不合适|调整pushDepth参数（建议 15-25）

## 🛠️ 运维命令参考
### Mac 服务端运维
（bash）
```bash

pkill -f server.py
```
